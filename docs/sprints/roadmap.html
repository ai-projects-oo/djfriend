Workflow Rules

Applies to every sprint and every ticket.

  • Sprint branch: git checkout main && git checkout -b sprint_XX
  • Each ticket: git checkout -b ticket/XX-slug off the sprint branch
  • Ticket merges back into sprint branch
  • No direct commits to main
Before merging sprint → main:
  1. npm run build — 0 errors
  2. npm run lint — 0 new errors
  3. npm test — all pass
  4. Run Opus code review agent — fix all findings
  5. User manual QA → explicit "ready to merge" approval
  6. git checkout main && git merge sprint_XX --no-ff && git push
SPRINT A
Post-Set Retrospective Score ✓ Complete — pending manual QA

Show a 0–100 quality badge on every History entry. Computed from data already on each SetTrack — no new analysis needed.

Key Facts

  • SetTrack already carries harmonicWarning, targetEnergy, energy, bpm
  • Score badge renders next to the mini energy curve in HistoryTab.tsx
  • All logic is pure — lives in app/lib/setScore.ts

Tickets

ticket/A1

app/lib/setScore.ts (new, pure)

Adds SetScore interface to types and computeSetScore(tracks) function.

Formula: (1-harmonicRate)×40 + (1-avgEnergyError)×35 + bpmSmoothness×25. Returns null for <2 tracks. Denominator for harmonic rate is n-1 transitions.

ticket/A2

app/components/HistoryTab.tsx

Renders score pill: green ≥80, amber ≥60, red <60. Tooltip shows component breakdown. aria-label for accessibility.

ticket/A3

test/set-score.test.ts (new)

11 test cases covering edge cases, denominator correctness, and clamping.

Pre-Merge Checklist

  • build, lint, test clean
  • Opus code review run + findings fixed
  • Manual QA: History tab shows badges on existing entries

SPRINT 9
Local AI Improvements Depends on: sprint_8 merged

Three self-contained AI upgrades — all local, no API keys, no cloud. Each ticket is independent and can ship separately.

Background

  • Sprint 8 shipped: ML vocal detection (voice_instrumental-msd-musicnn-1, ~2 MB), semantic tagging, harmonic move suggestions (+2 / −5 in key tooltip)
  • Remaining gaps identified: missing genre on local files, transition reasoning not shown in UI, set curve drawn manually every time

Tickets

ticket/9-1

Genre inference from audio (ML)

Use Essentia genre_rosamerica-msd-musicnn-1 (~2 MB, same MusiCNN pipeline as vocal detection) to label local files that have no genre tag. Labels: electronic, jazz, hip-hop, classical, rock, pop, soul, country, reggae, blues. Written to results.json as inferredGenre — never overwrites an existing ID3 genre tag.

  • New file: src/genre-detector.ts — mirrors vocal-detector.ts structure
  • Called from src/api.ts after analyzeAudio when genres.length === 0
  • Add inferredGenre?: string to app/types/index.ts Song interface
  • Show inferred genre in track row with a faint italic style (distinct from tag genres)
  • Feed into deriveSemanticTags so untagged library gets correct vibe/venue tags

Model: https://essentia.upf.edu/models/classifiers/genre_rosamerica/genre_rosamerica-msd-musicnn-1/ — same download-on-first-use pattern as vocal model

ticket/9-2

Full transition reasoning in UI

selectionReason already exists on SetTrack and is partially populated by the set generator. Surface it fully.

  • Set generator: populate all reason fields — harmonic score label, BPM delta, energy delta vs target, semantic tag match
  • TrackRow.tsx: expand the existing "AI tags" row toggle to also show selectionReason bullets when present
  • No new component needed — add a "Why chosen?" section inside the existing expandable row
ticket/9-3

Smart energy curve presets

Auto-fill the energy curve from the library's actual BPM/energy distribution. No ML needed — pure statistics over the loaded song pool.

  • Compute percentile breakpoints (p25/p50/p75 energy) from the loaded library
  • Four presets: Warm-up (low → mid), Peak set (mid → high → sustain), Closing (high → low), Rollercoaster (alternating)
  • Preset picker: 4 small buttons above the curve editor, replacing the current blank default
  • Pure function in app/lib/curvePresets.ts — no side effects, easy to test
  • Tests: assert each preset produces a valid CurvePoint[] with monotonic x values and y in [0,1]

  • Inferred genre shown in UI with italic style, not confused with tag genres
  • selectionReason bullets visible when expanding a track row
  • Curve presets produce sensible curves on a test library
  • Manual QA approval before merge
